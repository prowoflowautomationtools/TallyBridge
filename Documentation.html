TallyBridge Documentation

Your complete guide to using the Bank Statement Clearance Tool

Back to Application

Step 1: Getting Started on the Dashboard

When you first open TallyBridge, you'll land on the Dashboard, which presents two main options:

  • Start with New Bank Statement: Choose this to begin a new reconciliation process from scratch.
  • Continue Previous Work: Select this to load a previously saved batch and continue where you left off.

To start a new batch, you will be prompted to enter the Tally Company Name for which you are clearing transactions. This name is used for reference throughout the process.

Step 2: Import Tally Masters

Next, you need to provide the application with a list of your existing ledgers from Tally. This allows TallyBridge to know which ledgers are available for mapping.

How to Export Masters from Tally:

  1. In "Gateway of Tally", press Alt + E (Export).
  2. Select Masters and press Enter.
  3. Press C (Configure).
  4. Set "File Format" to ASCII (Comma delimited).
  5. Set the Folder Path and File Name.
  6. Press Ctrl + A to save the configuration.
  7. Press E (Send) to export the .txt file.

Once you have the .txt file, upload it to the application.

Step 3: Import Your Bank Statement

The application requires your bank statement data in a specific Excel format.

  1. Click the Download Template button to get the standard .xlsx file.
  2. Open the template and fill it with your bank statement details.
    • Note on Debit/Credit: The template follows standard bank statement conventions. Credit means deposits/receipts, and Debit means withdrawals/payments. This applies to all account types (Savings, Current, CC, OD).
  3. Save the file and upload it back into the TallyBridge application.
  4. You will be asked to select the corresponding Bank Account Ledger from the list of masters you imported earlier and You want to create a New Ledger then you can create it as well. This links the statement to the correct account in Tally.

Step 4: Clearing Transactions

After importing, you'll be taken to the "Clearing Page," where all your bank transactions are displayed in a table. Here's how to process them:

Ledger Mapping

For each transaction row, use the dropdown in the Ledger Mapping column to select the appropriate Tally ledger that corresponds to the transaction's particulars.

Creating New Ledgers

If a required ledger doesn't exist in the dropdown, click the Create Ledger button. A modal will appear where you can enter the new ledger's name, Tally group, and opening balance. Once created, it will be available for mapping.

Bulk Mapping

To speed up the process, you can map multiple transactions at once.

  1. Use the checkboxes in the first column to select multiple transactions.
  2. The Bulk Map button will become active, showing the number of selected items.
  3. Click it, choose the correct ledger from the modal, and click "Apply Mapping" to assign that ledger to all selected transactions.

Filtering and Sorting

Use the filter row at the top of the table to narrow down transactions by date, particulars, amount, and more. You can also click on sortable column headers (like Date, Debit, or Credit) to organize the data.

Step 5: Finalizing Transactions

Once you have mapped ledgers to the transactions you wish to finalise, you need to move them to the next stage.

  1. Click the Push Mapped to Finalise button. This action takes all transactions that have a ledger mapped and moves them to the "Finalise Page."
  2. To navigate to this page, click the View Finalise Page button.

Step 6: Review and Export

The "Finalise Page" is the last step before exporting your data for Tally. It shows a summary of the vouchers that will be created.

Pushing Back to Clearing

If you notice a mistake or need to change a ledger mapping, select the incorrect transaction(s) using the checkboxes and click the Push Back to Clearing button. This will move the selected items back to the previous page for re-mapping.

Generating XML Files

This is the most important part of the process. You must generate and import two separate XML files in the correct order.

  1. Generate Masters.xml: Click this button first. It creates an XML file containing all the new ledgers you created during this session.
  2. Generate Vouchers.xml: Click this button second. It creates an XML file containing all the cleared bank transactions ready to be imported as vouchers into Tally.

Step 7: How to Import XML Files into Tally

After downloading the two XML files from TallyBridge, follow these steps to import them into Tally.

Important: Always take a backup of your Tally company data before performing any import.

Part 1: Import Masters

  1. Place the generated Masters.xml file in an easily accessible folder on your computer (e.g., C:\Tally_Import).
  2. Open Tally and select the company you are working with.
  3. From the "Gateway of Tally", navigate to Import Data → Masters.
  4. For the "File Path", enter the full path to your file (e.g., C:\Tally_Import\Masters.xml).
  5. Set the "Behaviour" to Modify with new data. This will add your new ledgers without affecting existing ones.
  6. Press Enter to begin the import.

Part 2: Import Vouchers

  1. After the masters have been successfully imported, navigate back to the "Gateway of Tally".
  2. Go to Import Data → Vouchers.
  3. For the "File Path", enter the full path to your Vouchers.xml file (e.g., C:\Tally_Import\Vouchers.xml).
  4. Press Enter to import all the cleared transactions.
  5. Once complete, you can verify the imported data by checking the Day Book or relevant ledger reports in Tally.
